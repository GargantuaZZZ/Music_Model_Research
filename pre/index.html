  • 49 位世界知名钢琴家

05

Metrics

评价指标

音频生成质量 4 项

FAD (Fréchet Audio Distance)

音频生成质量

流程

真实音频 → 预训练音频模型 → embedding 分布

生成音频 → 同一个预训练音频模型 → embedding 分布

比较两个分布的距离

关注内容

  • 音质是否自然
  • 音色是否接近真实数据
  • 噪声、失真、伪影是否多
  • 整体声学分布是否像真实录音

KL (Kullback-Leibler Divergence)

音频生成质量

DKL(P ∥ Q) = ∑x P(x) log P(x) Q(x)

其中

  • P：真实分布 / 目标分布
  • Q：模型分布 / 近似分布

KL 可以比较 genre、mood、instrument、audio event、embedding / feature statistics 的分布。如果 KL 小，说明生成音乐在这些语义或声学类别上的分布更接近真实数据。

Spectrogram distance

音频生成质量

流程

真实音频 → 时频图

生成音频 → 时频图

比较两个频谱间的距离

关注内容

  • 每个时间点的频率能量是否相近
  • 泛音结构是否相近
  • 音符起音/衰减是否相近
  • 高频细节是否缺失
  • 是否有多余噪声或频带能量
要求 生成音频与真实音频时间对齐

Chroma distance

音频生成质量

把音高压缩到 12 个 pitch class，比较生成音频和真实音频的 chroma 特征差异

关注内容

  • 音高类别是否正确
  • 和声是否接近
  • 和弦结构是否一致
  • 是否弹错音或漏掉重要音高
MIDI生成质量 3 项

Pitch Class Entropy

MIDI生成质量

目标：衡量音高类别分布的丰富度。

Pitch class 映射

先把所有非鼓音符映射到 12 个 pitch class： pc = pitch mod 12

概率与熵

pi = ni Σj=112 nj
H = −Σi=112 pi log pi

Scale Consistency

MIDI生成质量

目标：衡量生成音乐是否符合某个音阶/调式。

对每首 MIDI，遍历可能的调/音阶，比如 12 个 major + 12 个 minor，找一个最能解释这首曲子的 scale。

候选 scale 评分

对于某个候选 scale S，计算有多少音符的 pitch class 落在这个 scale 内：

SC(S) = #{notes: pc(note) ∈ S} #{notes}

取最大值

SC = maxS SC(S)

Groove Consistency

MIDI生成质量

目标：衡量节奏律动是否稳定。

常见实现是把每个小节切成固定网格，比如 16 个 step，并对每个小节构造一个 onset pattern。

Onset pattern 示例

bar 1

[1,0,0,1, 0,0,1,0, ...]

bar 2

[1,0,0,1, 0,0,1,0, ...]

meaning

1 表示该位置有 onset，0 表示没有

然后比较不同小节之间节奏 pattern 的相似度。常见可以用 Hamming similarity / cosine similarity / pairwise distance。

距离越小，consistency 越高。

音乐理解能力 7 项

ROUGE

音乐理解能力

ROUGE-1

看单个词的重叠。

reference

the song is about lost love

candidate

this song describes love and loss

重叠词

song, love

ROUGE-2

看连续两个词的重叠。

ROUGE-L

看最长公共子序列。它不要求词必须连续，但要求顺序一致。

METEOR

音乐理解能力

METEOR 不只看完全相同的词，还会考虑词形变化、词干、同义词、词序惩罚。

公式

Precision = matched candidate words candidate words
Recall = matched reference words reference words
METEOR = Fmean × (1 − penalty)

其中 penalty 为惩罚项，用于惩罚词序太乱、匹配不连续的情况。

BERT-Score

音乐理解能力

BERT-Score 不直接比较词面是否相同，而是用 BERT 这类预训练语言模型把每个 token 表示成上下文向量，然后比较向量相似度。

流程

reference → BERT → token embeddings

candidate → BERT → token embeddings

计算 candidate token 和 reference token 的余弦相似度

对于 candidate 中每个 token，找 reference 中最相似的 token；反过来也做一次。

得到

PBERT

表示 candidate 中的词有多少能在 reference 中找到语义相近的对应。

RBERT

表示 reference 中的词有多少被 candidate 覆盖。

FBERT = 2 · PBERT RBERT PBERT + RBERT

MRR (Mean Reciprocal Rank)

音乐理解能力

用生成的 interpretation 帮助检索歌曲。

公式

MRR = 1 m Σi=1m 1 ki

其中 ki 是第 i 个 query 的正确歌曲在检索结果里的排名。

MuChoMusic

音乐理解能力

包含 1,187 道选择题，覆盖 644 段音乐

题目分为 Knowledge 问题（考察模型是否能识别音乐里的基本客观属性）和 Reasoning 问题（考察模型能否结合多个音乐信息，推断音乐功能场景）

评价指标

1. Accuracy
Accuracy = # correct answers # total questions × 100%
2. Instruction Following Rate (IFR)
IFR = # outputs matching one valid option # total outputs × 100%

PI (Perceptual Index)

音乐理解能力

公式

PI(q,Y,D) = ptext(D|q) ptext(Y|q)

其中

  • q：问题文本
  • Y：所有候选答案
  • D：错误干扰项集合
  • ptext：text-only LLM 在只看文本时给出的概率

解释

PI 越高：text-only 模型越容易选错，说明这道题越需要音频感知。

PI 越低：text-only 模型越容易靠文本猜对，说明这道题不太能检验音频理解。

valence-arousal

音乐理解能力

valence = 情感效价，表示情绪是偏正面还是负面；

arousal = 唤醒度 / 激活度，表示情绪能量强不强；

高 valence + 高 arousal 兴奋、快乐、激动
高 valence + 低 arousal 平静、舒适、放松
低 valence + 高 arousal 愤怒、焦虑、恐惧
低 valence + 低 arousal 悲伤、沮丧、无聊
音源分离效果 3 项

SDR (Signal-to-Distortion Ratio)

音源分离效果

SDR 衡量的是：分离/生成出来的音频和真实目标音频相比，失真有多小。

公式

SDR = 10 log10 ||starget||2 ||starget − ŝ||2

其中

  • starget：真实目标声源，比如真实人声、真实钢琴
  • ŝ：模型估计/分离出来的声源
  • starget − ŝ：误差或失真
  • 单位通常是 dB

SIR (Signal-to-Interference Ratio)

音源分离效果

SIR 衡量的是：目标声源里混进了多少其他声源。

公式

SIR = 10 log10 ||starget||2 ||einterf||2

其中

  • starget：目标源成分
  • einterf：其他源泄漏进来的干扰

SAR (Signal-to-Artifacts Ratio)

音源分离效果

SAR 衡量的是：分离结果里有多少算法产生的伪影。

伪影 (artifacts) 指的是原本音频里没有、由模型处理造成的奇怪声音。

公式

SAR = 10 log10 ||starget + einterf||2 ||eartif||2

其中

  • eartif：算法产生的伪影误差
音频对齐误差 2 项

Measure Transfer

音频对齐误差

场景：只有一个版本的音频有人工小节标注

S = score / 乐谱

A1 = 要评估的某个音频版本

A2 = WK64 音频版本，有人工小节标注

两条路径

把同一个小节位置用两条路径转到 A1 的时间轴上：

路径 1：从 score 转到 A1
𝓜S→A1(tS)
路径 2：从 WK64 音频转到 A1
𝓜A2→A1(tA2)

绝对时间误差

ε = |𝓜S→A1(tS) − 𝓜A2→A1(tA2)|

Note Onset Transfer

音频对齐误差

场景：有音频对应的详细 MIDI 信息

S = score

A1 = 一个音频版本

A2 = 另一个音频版本

score-audio alignment

先分别做 score-audio alignment：

𝓜S→A1,  𝓜S→A2

把 score 里的 note onset tS 映射到两个音频时间轴上。

audio-audio alignment

𝓜A1→A2

路径误差

比较两条路径到达 A2 后的位置差：

ε = |𝓜S→A2(tS) − 𝓜A1→A2(𝓜S→A1(tS))|

06

Challenges

难点与挑战

数据集

1

缺少某种类型的数据集

比如特定流派的音乐，可分离各音源的音乐，高质量、对齐的 MIDI 等。

2

数据集质量不高，缺少标注

很多数据集的音频质量不高或有背景音；数据集没有或缺少流派、情绪、说明等标注。

3

数据集缺乏统一标准

不同论文常常使用不同训练集和测试集，有些还用专有数据集，导致很难判断真正的 SOTA。

模型

1

依赖预训练模型性能

许多使用的预训练模型通常不是专门为音乐或其细分领域训练的，可能需要再训练。

2

可控性和定制化不足

大多数现有模型像黑箱，用户很难精确控制乐器、节奏、情绪曲线、音乐结构、和声等，严重限制了实际应用。

评价指标

1

评价指标不统一

不同论文的指标难以横向比较，难以判断真正的 SOTA。

2

评价指标不可靠

FAD、FD、KL 等多基于通用音频特征，不一定适合音乐质量评价。MIDI 统计指标只能看局部统计，和人类偏好相关性低。很多指标不能很好衡量叙事、情绪、结构、审美。

音频 or MIDI

MIDI

优点
  • 可控性强
  • 可编辑
  • 容易加入和弦、旋律、节奏、乐器等控制
  • 可以生成较长结构
缺点
  • 数据规模小
  • 多样性有限
  • 表现力依赖 soundfont 或后续渲染
  • 音色和真实演奏质感弱

音频

优点
  • 声音表现力强
  • 可直接生成真实音频
  • 可利用大规模音频数据
缺点
  • 可控性弱
  • 计算成本高
  • 生成时长通常较短
  • 很难精细编辑

MIDI -> 音频

1

对不同声学环境和音色的泛化能力

数据集种类有限，模型架构设计限制，都会影响模型在不同乐器音色、录音条件、混响环境和演奏风格下的泛化表现。

2

EPR 和 EPS 模型差异导致的损失

EPR (expressive performance rendering) 将乐谱转换为 MIDI，EPS (expressive performance synthesis) 将 MIDI 转换为音频。二者处理和表示 MIDI 信号的方式不同，可能影响最终合成质量。

视频 -> 音频

1

领域还处在早期阶段

该领域还没有形成成熟的工业应用和统一研究范式。模型不仅要生成好听的音乐，还要让音乐和视觉内容在语义、情绪、节奏、时间结构上匹配。

2

视觉与音乐之间的对应关系很难建模

视觉和音乐不是天然一一对应的，模型很难判断“唯一正确”的音乐是什么。

07

Applications

可能的应用

7.1

MIDI + 音频 -> 音频：参考音频驱动的快速音色尝试

传统软音源通常需要用户手动选择音色库，并调整大量参数，例如乐器类型、力度响应、包络、混响、空间感、音色明暗、演奏法等。对于非专业制作人来说，这个过程门槛较高，而且试错成本很大。

MIDI + 音频 -> 音频的模型可以提供一种更直观的工作方式：用户输入 MIDI 作为旋律、节奏和音高结构，再输入一段参考音频作为音色、演奏风格或声音质感参考，模型直接生成接近该参考风格的音频。这样用户不需要理解复杂的合成器参数，只需要通过“给一段参考声音”来快速尝试不同音色。

应用场景

  • 编曲时快速试听同一段 MIDI 在不同音色下的效果。
  • 用参考钢琴、人声、弦乐或合成器音色生成更接近目标风格的音频。
  • 给不会调软音源参数的创作者提供低门槛音色设计工具。
  • 在游戏、短视频、广告配乐中快速生成多版音色 demo。
  • 把 MIDI 草稿快速渲染成更真实、更有表现力的音频，用于创作预览。

它相比传统软音源的优势不是完全替代专业音源，而是降低早期创作阶段的试错成本：从“调参数”变成“找参考”。

7.2

理解 -> MIDI：文本控制的可编辑音乐草稿

理解 -> MIDI 可以把自然语言描述、标签或情绪设定转成可编辑 MIDI。相比直接生成音频，MIDI 更容易修改旋律、和弦、节奏、速度和结构，因此适合作为创作中间层。

应用场景

  • 输入“轻快、适合 vlog、带一点 funk 的钢琴伴奏”，生成 MIDI 草稿。
  • 根据情绪标签生成旋律或和弦进行。
  • 为不会乐理的用户生成可编辑音乐片段。
  • 给专业制作人提供可修改的灵感素材。
  • 先生成 MIDI，再接 MIDI -> 音频模型生成真实音色。

它的优势是可控和可编辑，适合作为音乐创作软件里的“草稿生成器”。

7.3

音频 -> 理解 -> 视频 / 文本：音乐内容解释与 MV 自动生成

音频 -> 理解模型不仅可以把音乐转成标签，还可以进一步分析音乐的情绪、风格、主题、歌词含义、段落结构、高潮位置、乐器使用、节奏变化和整体氛围。这类结构化理解结果可以继续接到下游生成模型，也可以直接作为面向听众的解释内容。

7.3.1 音乐 MV 自动生成

相比直接把整首歌丢给视频生成模型，先做音乐理解有两个好处：可以让视频内容更贴合音乐结构，例如主歌画面更克制，副歌画面更强烈，bridge 部分可以加入转场或情绪变化；也可以让视频风格更贴合音乐语义，例如模型先判断歌曲是“忧郁、低唤醒、怀旧感”，再生成慢镜头、低饱和度、空旷场景。如果是“兴奋、高唤醒、电子舞曲”，则生成高运动量、强光效、快切镜头。

  • 给独立音乐人生成 MV 初稿。
  • 给短视频平台生成音乐可视化内容。
  • 给音乐播放器生成动态封面、歌词视频或循环背景。
  • 给影视配乐生成概念视觉预览。
  • 根据歌曲段落自动剪辑不同视觉片段。

这里的核心价值是：音乐理解模型把“声音”变成可控的视觉生成条件，让视频生成不只是随机配画面，而是更有依据地跟随音乐。

7.3.2 听众听歌的背景参考

音乐理解还可以作为听众理解音乐的辅助信息。很多时候，听众听到一首歌，只能模糊感受到“好听”“悲伤”“很燃”，但不一定知道它为什么会产生这种感觉。音频 -> 理解模型可以把音乐中的结构、情绪和风格线索解释出来，作为听歌时的背景参考。

  • 音乐播放器中的“歌曲解读”功能。
  • 给听众解释一首歌的情绪、结构、风格和听感来源。
  • 帮助用户发现相似音乐，而不是只依赖协同过滤推荐。
  • 给音乐学习者提供听歌分析，例如和声、节奏、编曲特点。
  • 为歌单生成更有内容的信息，例如“为什么这些歌适合深夜听”。
  • 给短视频创作者解释某首歌适合什么画面和情绪场景。

这类应用不一定直接生成新内容，但能提升听众对音乐的理解和参与感。它更像是“音乐导览”或“AI 乐评助手”：把模型听到的音乐信息，转化成普通听众能理解的语言。

7.4

视频 -> 音频 / MIDI：自动配乐与音画同步

视频到音乐可以根据画面内容、情绪、节奏和场景生成配乐。相比普通 text-to-music，它多了视觉条件，因此更适合短视频、广告、游戏 cutscene、影视片段等场景。

应用场景

  • 给短视频自动生成背景音乐。
  • 根据镜头节奏生成匹配的音乐节拍。
  • 为电影片段生成情绪一致的 soundtrack。
  • 为游戏场景生成动态配乐。
  • 生成 MIDI 草稿后再由音频模型渲染成真实音乐。

难点在于视频和音乐没有唯一对应关系，所以更现实的产品形态可能不是“一键生成最终配乐”，而是生成多版候选，让用户选择和微调。

7.5

音频 + 参考音频 / 参考视频 -> 分离：智能扒带与二创

分离模型可以根据参考音频或参考视频，从混合音乐中提取目标声源。相比传统固定类别分离，比如只分人声、鼓、贝斯、其他，这类方法更灵活，可以指定“我想要这个声音”。

应用场景

  • 从歌曲中分离特定乐器，辅助扒带和练习。
  • 提取人声、钢琴、吉他等声部，用于 remix 或二创。
  • 根据演奏视频中的乐器画面，分离对应音轨。
  • 音乐教学中单独放大某个乐器声部。
  • 老录音修复中提取目标声源，减少背景干扰。

这类应用的产品价值很强，因为它直接对应真实音乐制作和学习需求。
