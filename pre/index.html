04

Datasets

数据集

ATEPP (Automatically Transcribed Expressive Piano Performance)

Transcribed MIDI

  • 11,742 首/轨钢琴演奏
  • 约 1007 小时音乐
  • 49 位世界知名钢琴家
  • 25 位西方古典作曲家
  • 1580 个乐章/曲目单位
  • 主要覆盖从巴洛克到现代时期的标准古典钢琴独奏曲目
  • 约 43% 的轨道配有 MusicXML 乐谱，共 319 个乐章，对应 5124 条演奏
  • 强调同一曲目的多演奏版本。1580 个乐章中有 44% 拥有超过 5 个演奏版本，约 12% 拥有超过 15 个版本。使用 MIDI 的 onset、offset、velocity、pitch 等音符级特征，可学习到个人演奏风格信息

MAESTRO (MIDI and Audio Edited for Synchronous TRacks and Organization)

钢琴音频-Recorded MIDI 对齐

  • 1184 段钢琴演奏
  • 约 430 首/个作品
  • 172.3 小时音频
  • 约 618 万个音符
  • 音频大小约 102.8 GB
  • 音频质量为 44.1-48 kHz、16-bit PCM stereo
  • 包含真实钢琴音频，对齐 MIDI，力度与踏板等信息
  • 包含作曲家、曲名、演奏年份等元数据
  • 官方 train/validation/test 划分

OSSL (Open Screen Soundtrack Library)

音视频，影视配乐

  • 来自 299 部公版电影的 736 个电影片段
  • 总时长约 36.5 小时，平均片段长度约 178.47 秒
  • 电影视频片段
  • 对应的 soundtrack
  • 电影元数据，如标题、上映时间、类型
  • 人工标注的 mood 情绪标签

MUSIC

音频与对应演奏视频

  • 714 个 YouTube 视频，其中 565 个 solo 视频，149 个 duet 视频
  • 平均视频时长约 2 分钟
  • 覆盖 11 类乐器
  • 随后的 MUSIC-21 扩展了视频数量和乐器种类

GlobalMood

音频，多模态

  • 1,180 首歌曲，共来自 59 个国家，每个国家采样 20 首歌
  • 每首歌使用 15 秒音频片段
  • 共 2,519 名参与者，来自 5 个语言/文化地区
  • 最终收集 988,925 条情绪/心境评分
  • 不预设情绪词表，而是让参与者用自己的母语自由写出音乐传达的情绪/心境词，明显提升非英语语境下的情绪评价效果

Music4All

音频，多模态

  • 109,269 首歌曲
  • 15,602 个匿名用户
  • 用户的听歌历史
  • 每首歌的 30 秒音频片段
  • 歌词，语言信息，标签，风格/流派
  • Spotify 元数据与音频属性

05

Metrics

评价指标

音频生成质量 3 项

FAD (Fréchet Audio Distance)

音频生成质量

流程

真实音频 → 预训练音频模型 → embedding 分布

生成音频 → 同一个预训练音频模型 → embedding 分布

比较两个分布的距离

关注内容

  • 音质是否自然
  • 音色是否接近真实数据
  • 噪声、失真、伪影是否多
  • 整体声学分布是否像真实录音

Spectrogram distance

音频生成质量

流程

真实音频 → 时频图

生成音频 → 时频图

比较两个频谱间的距离

关注内容

  • 每个时间点的频率能量是否相近
  • 泛音结构是否相近
  • 音符起音/衰减是否相近
  • 高频细节是否缺失
  • 是否有多余噪声或频带能量
要求 生成音频与真实音频时间对齐

Chroma distance

音频生成质量

把音高压缩到 12 个 pitch class，比较生成音频和真实音频的 chroma 特征差异

关注内容

  • 音高类别是否正确
  • 和声是否接近
  • 和弦结构是否一致
  • 是否弹错音或漏掉重要音高
MIDI生成质量 3 项

Pitch Class Entropy

MIDI生成质量

目标：衡量音高类别分布的丰富度。

Pitch class 映射

先把所有非鼓音符映射到 12 个 pitch class： pc = pitch mod 12

概率与熵

pi = ni Σj=112 nj
H = −Σi=112 pi log pi

Scale Consistency

MIDI生成质量

目标：衡量生成音乐是否符合某个音阶/调式。

对每首 MIDI，遍历可能的调/音阶，比如 12 个 major + 12 个 minor，找一个最能解释这首曲子的 scale。

候选 scale 评分

对于某个候选 scale S，计算有多少音符的 pitch class 落在这个 scale 内：

SC(S) = #{notes: pc(note) ∈ S} #{notes}

取最大值

SC = maxS SC(S)

Groove Consistency

MIDI生成质量

目标：衡量节奏律动是否稳定。

常见实现是把每个小节切成固定网格，比如 16 个 step，并对每个小节构造一个 onset pattern。

Onset pattern 示例

bar 1

[1,0,0,1, 0,0,1,0, ...]

bar 2

[1,0,0,1, 0,0,1,0, ...]

meaning

1 表示该位置有 onset，0 表示没有

然后比较不同小节之间节奏 pattern 的相似度。常见可以用 Hamming similarity / cosine similarity / pairwise distance。

距离越小，consistency 越高。

音乐理解能力 7 项

ROUGE

音乐理解能力

ROUGE-1

看单个词的重叠。

reference

the song is about lost love

candidate

this song describes love and loss

重叠词

song, love

ROUGE-2

看连续两个词的重叠。

ROUGE-L

看最长公共子序列。它不要求词必须连续，但要求顺序一致。

METEOR

音乐理解能力

METEOR 不只看完全相同的词，还会考虑词形变化、词干、同义词、词序惩罚。

公式

Precision = matched candidate words candidate words
Recall = matched reference words reference words
METEOR = Fmean × (1 − penalty)

其中 penalty 为惩罚项，用于惩罚词序太乱、匹配不连续的情况。

BERT-Score

音乐理解能力

BERT-Score 不直接比较词面是否相同，而是用 BERT 这类预训练语言模型把每个 token 表示成上下文向量，然后比较向量相似度。

流程

reference → BERT → token embeddings

candidate → BERT → token embeddings

计算 candidate token 和 reference token 的余弦相似度

对于 candidate 中每个 token，找 reference 中最相似的 token；反过来也做一次。

得到

PBERT

表示 candidate 中的词有多少能在 reference 中找到语义相近的对应。

RBERT

表示 reference 中的词有多少被 candidate 覆盖。

FBERT = 2 · PBERT RBERT PBERT + RBERT

MRR (Mean Reciprocal Rank)

音乐理解能力

用生成的 interpretation 帮助检索歌曲。

公式

MRR = 1 m Σi=1m 1 ki

其中 ki 是第 i 个 query 的正确歌曲在检索结果里的排名。

MuChoMusic

音乐理解能力

包含 1,187 道选择题，覆盖 644 段音乐

题目分为 Knowledge 问题（考察模型是否能识别音乐里的基本客观属性）和 Reasoning 问题（考察模型能否结合多个音乐信息，推断音乐功能场景）

评价指标

1. Accuracy
Accuracy = # correct answers # total questions × 100%
2. Instruction Following Rate (IFR)
IFR = # outputs matching one valid option # total outputs × 100%

PI (Perceptual Index)

音乐理解能力

公式

PI(q,Y,D) = ptext(D|q) ptext(Y|q)

其中

  • q：问题文本
  • Y：所有候选答案
  • D：错误干扰项集合
  • ptext：text-only LLM 在只看文本时给出的概率

解释

PI 越高：text-only 模型越容易选错，说明这道题越需要音频感知。

PI 越低：text-only 模型越容易靠文本猜对，说明这道题不太能检验音频理解。

valence-arousal

音乐理解能力

valence = 情感效价，表示情绪是偏正面还是负面；

arousal = 唤醒度 / 激活度，表示情绪能量强不强；

高 valence + 高 arousal 兴奋、快乐、激动
高 valence + 低 arousal 平静、舒适、放松
低 valence + 高 arousal 愤怒、焦虑、恐惧
低 valence + 低 arousal 悲伤、沮丧、无聊
音源分离效果 3 项

SDR (Signal-to-Distortion Ratio)

音源分离效果

SDR 衡量的是：分离/生成出来的音频和真实目标音频相比，失真有多小。

公式

SDR = 10 log10 ||starget||2 ||starget − ŝ||2

其中

  • starget：真实目标声源，比如真实人声、真实钢琴
  • ŝ：模型估计/分离出来的声源
  • starget − ŝ：误差或失真
  • 单位通常是 dB

SIR (Signal-to-Interference Ratio)

音源分离效果

SIR 衡量的是：目标声源里混进了多少其他声源。

公式

SIR = 10 log10 ||starget||2 ||einterf||2

其中

  • starget：目标源成分
  • einterf：其他源泄漏进来的干扰

SAR (Signal-to-Artifacts Ratio)

音源分离效果

SAR 衡量的是：分离结果里有多少算法产生的伪影。

伪影 (artifacts) 指的是原本音频里没有、由模型处理造成的奇怪声音。

公式

SAR = 10 log10 ||starget + einterf||2 ||eartif||2

其中

  • eartif：算法产生的伪影误差
音频对齐误差 2 项

Measure Transfer

音频对齐误差

场景：只有一个版本的音频有人工小节标注

S = score / 乐谱

A1 = 要评估的某个音频版本

A2 = WK64 音频版本，有人工小节标注

两条路径

把同一个小节位置用两条路径转到 A1 的时间轴上：

路径 1：从 score 转到 A1
𝓜S→A1(tS)
路径 2：从 WK64 音频转到 A1
𝓜A2→A1(tA2)

绝对时间误差

ε = |𝓜S→A1(tS) − 𝓜A2→A1(tA2)|

Note Onset Transfer

音频对齐误差

场景：有音频对应的详细 MIDI 信息

S = score

A1 = 一个音频版本

A2 = 另一个音频版本

score-audio alignment

先分别做 score-audio alignment：

𝓜S→A1,  𝓜S→A2

把 score 里的 note onset tS 映射到两个音频时间轴上。

audio-audio alignment

𝓜A1→A2

路径误差

比较两条路径到达 A2 后的位置差：

ε = |𝓜S→A2(tS) − 𝓜A1→A2(𝓜S→A1(tS))|

06

Challenges

难点与挑战

数据集

1

缺少某种类型的数据集

比如特定流派的音乐，可分离各音源的音乐，高质量、对齐的 MIDI 等。

2

数据集质量不高，缺少标注

很多数据集的音频质量不高或有背景音；数据集没有或缺少流派、情绪、说明等标注。

3

数据集缺乏统一标准

不同论文常常使用不同训练集和测试集，有些还用专有数据集，导致很难判断真正的 SOTA。

模型

1

依赖预训练模型性能

许多使用的预训练模型通常不是专门为音乐或其细分领域训练的，可能需要再训练。

2

可控性和定制化不足

大多数现有模型像黑箱，用户很难精确控制乐器、节奏、情绪曲线、音乐结构、和声等，严重限制了实际应用。

评价指标

1

评价指标不统一

不同论文的指标难以横向比较，难以判断真正的 SOTA。

2

评价指标不可靠

FAD、FD、KL 等多基于通用音频特征，不一定适合音乐质量评价。MIDI 统计指标只能看局部统计，和人类偏好相关性低。很多指标不能很好衡量叙事、情绪、结构、审美。

音频 or MIDI

MIDI

优点
  • 可控性强
  • 可编辑
  • 容易加入和弦、旋律、节奏、乐器等控制
  • 可以生成较长结构
缺点
  • 数据规模小
  • 多样性有限
  • 表现力依赖 soundfont 或后续渲染
  • 音色和真实演奏质感弱

音频

优点
  • 声音表现力强
  • 可直接生成真实音频
  • 可利用大规模音频数据
缺点
  • 可控性弱
  • 计算成本高
  • 生成时长通常较短
  • 很难精细编辑

多模态（视频）

1

领域还处在早期阶段

该领域还没有形成成熟的工业应用和统一研究范式。模型不仅要生成好听的音乐，还要让音乐和视觉内容在语义、情绪、节奏、时间结构上匹配。

2

视觉与音乐之间的对应关系很难建模

视觉和音乐不是天然一一对应的，模型很难判断“唯一正确”的音乐是什么。
